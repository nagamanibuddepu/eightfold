# Eightfold Candidate Transformer

Deterministic multi-source candidate data transformer for the Eightfold AI Engineering Intern assignment.

The project ingests structured sources (recruiter CSV, ATS JSON) and unstructured/semi-unstructured sources (resume PDF via pdfplumber, recruiter notes, GitHub profile URL via the public REST API), normalizes them, resolves identity, merges into one canonical candidate, records provenance and confidence, then optionally projects the canonical record into a runtime-configured output shape.

## Quick Start

```bash
python -m pip install -r requirements.txt
python -m pytest
```

Run the default canonical output:

```bash
python -m candidate_transformer.cli ^
  --recruiter-csv sample_data/recruiter_export.csv ^
  --ats-json sample_data/ats_candidate.json ^
  --notes-txt sample_data/recruiter_notes.txt ^
  --resume sample_data/resume.pdf ^
  --github-url https://github.com/nagamanibuddepu ^
  --github-cache sample_data/github_api_cache.json ^
  --run-date 2026-06-30
```

Write the default output:

```bash
python -m candidate_transformer.cli ^
  --recruiter-csv sample_data/recruiter_export.csv ^
  --ats-json sample_data/ats_candidate.json ^
  --notes-txt sample_data/recruiter_notes.txt ^
  --resume sample_data/resume.pdf ^

  --github-cache sample_data/github_api_cache.json ^
  --run-date 2026-06-30 ^

```

Run a custom projection:

```bash
python -m candidate_transformer.cli ^
  --recruiter-csv sample_data/recruiter_export.csv ^
  --ats-json sample_data/ats_candidate.json ^
  --notes-txt sample_data/recruiter_notes.txt ^
  --resume sample_data/resume.pdf ^
  --github-url https://github.com/nagamanibuddepu ^
  --github-cache sample_data/github_api_cache.json ^
  --config configs/custom_projection.json ^
  --run-date 2026-06-30 ^
  --output sample_outputs/custom_projection_output.json
```

On macOS/Linux, replace `^` line continuations with `\`.

## Pipeline

The code follows the required clean pipeline:

`Detect -> Parse -> Extract -> Normalize -> Identity Resolution -> Merge -> Confidence -> Provenance -> Canonical Candidate -> Projection -> Validation -> Output`

Important separation rules:

- Normalization is source-local and never depends on merged data.
- Projection is a read-only transform and never mutates the canonical model.
- Validation happens at three stages: config, canonical model, and projected output.

## Repository Layout

- `candidate_transformer/adapters/`: source adapters for CSV, ATS JSON, notes, resume PDF, and GitHub public REST API.
- `candidate_transformer/core/`: normalization, identity resolution, merge, confidence, and experience derivation.
- `candidate_transformer/projection/`: runtime projection config and canonical path reader.
- `candidate_transformer/validation/`: config, canonical, and output validators.
- `sample_data/`: assignment input files, including the candidate resume PDF, fictitious ATS/CSV/notes data, and one malformed CSV edge case.
- `sample_outputs/`: reproducible outputs generated by the CLI.
- `tests/`: focused unit and end-to-end tests.
- `docs/`: design, assumptions, requirement audit, and demo script.

## Design Choices

- Identity resolution is deterministic: normalized email first, then `normalized_full_name + E.164 phone` only when both are available.
- `candidate_id` is the first 16 hex characters of `sha256(identity_key)`, avoiding random UUIDs.
- Scalar merge uses source priority: resume, ATS JSON, recruiter CSV, recruiter notes, GitHub. GitHub contributes inferred evidence and does not establish identity or override claimed fields.
- Experience and education entries deduplicate on normalized company/title or institution keys, then merge missing fields from lower-priority sources.
- Phone arrays deduplicate E.164 variants that share the same national number, preferring the country hinted by source location.
- Missing or malformed sources become `_issues` instead of crashing the run.
- Confidence is simple and explainable: source reliability, agreement bonus, and conflict penalty.

## Outputs

- Default canonical output: `sample_outputs/default_output.json`
- Custom projection output: `sample_outputs/custom_projection_output.json`

## Tests

```bash
python -m pytest
```

Coverage focuses on:

- email, phone, country, and skill normalization
- resume PDF extraction via pdfplumber
- projection immutability
- end-to-end merge and custom projection
- malformed source graceful degradation
- validation failure paths for bad config/output shapes
- GitHub being supplemental evidence rather than an identity source

## Scope Notes

LinkedIn fetching is deliberately out of scope for this assignment submission. The GitHub adapter uses the public REST API first and accepts a cache fallback so demos/tests remain deterministic when the network is unavailable. See `docs/ASSUMPTIONS.md` for the full list.
