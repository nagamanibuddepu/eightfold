# Requirement Audit

## Assignment Requirements

| Requirement | Implementation |
| --- | --- |
| Structured source adapter | `candidate_transformer/adapters/csv_adapter.py`, `candidate_transformer/adapters/ats_json_adapter.py` |
| Unstructured source adapter | `candidate_transformer/adapters/notes_adapter.py`; GitHub profile URL via public REST API in `github_adapter.py` |
| Canonical model | `CANONICAL_TEMPLATE` in `candidate_transformer/core/merge.py` |
| Normalization | `candidate_transformer/core/normalize.py` for emails, phones, country, dates, links, skills |
| Deterministic identity resolution | `candidate_transformer/core/identity.py`; `candidate_transformer/pipeline.py` excludes GitHub from identity grouping |
| Merge engine | `candidate_transformer/core/merge.py` |
| Provenance | `provenance` entries created during merge for each populated value |
| Confidence engine | `candidate_transformer/core/confidence.py` |
| Configurable projection layer | `candidate_transformer/projection/projector.py`, `configs/custom_projection.json` |
| Validation | `candidate_transformer/validation/validators.py` validates config, canonical, and projected output |
| CLI | `candidate_transformer/cli.py` |
| Tests | `tests/test_normalize.py`, `tests/test_projection.py`, `tests/test_pipeline.py`, `tests/test_validation.py` |
| Sample data | `sample_data/` |
| Sample outputs | `sample_outputs/default_output.json`, `sample_outputs/custom_projection_output.json` |
| README | `README.md` |
| Design document | `docs/DESIGN.md`, generated PDF in `outputs/Nagamani_Buddepu_nagamani.buddepu.dev@example.com_Eightfold.pdf` |
| Demo script guidance | `docs/DEMO_SCRIPT.md` |
| Assumptions | `docs/ASSUMPTIONS.md` |

## Pipeline Audit

| Required Stage | Where It Happens |
| --- | --- |
| Detect | CLI input flags and `_detect_parse_extract` in `candidate_transformer/pipeline.py` |
| Parse | Adapter modules under `candidate_transformer/adapters/` |
| Extract | Adapter field mapping and conservative regex extraction |
| Normalize | `normalize_source_profile` in `candidate_transformer/core/normalize.py` |
| Identity Resolution | `choose_candidate_group` in `candidate_transformer/core/identity.py` |
| Merge | `merge_profiles` in `candidate_transformer/core/merge.py` |
| Confidence | `candidate_transformer/core/confidence.py` |
| Provenance | Merge helpers append field/source/method records |
| Canonical Candidate | Canonical dictionary returned by `merge_profiles` |
| Projection | `project_candidate` |
| Validation | `validate_config`, `validate_canonical`, `validate_projected_output` |
| Output | CLI stdout or `--output` file |

## Behavioral Requirements

| Requirement | Evidence |
| --- | --- |
| Deterministic | Candidate IDs use SHA-256 of identity key; arrays are sorted or priority ordered; tests use fixed `--run-date`. |
| Robust on malformed input | `sample_data/malformed_recruiter_export.csv` and `test_malformed_source_is_reported_not_fatal`. |
| Missing values are not invented | Normalizers return `None`; projection respects `on_missing`; adapters only map/extract observed data. |
| Provenance preserved | Canonical validation requires provenance for populated fields. |
| Output validates | CLI calls validation before writing. |
| Default and custom outputs work | Both sample output files are generated by CLI commands in `README.md`. |
| Projection does not mutate canonical | `test_projection_does_not_mutate_canonical`. |
| GitHub is not an identity source | `test_github_never_establishes_identity`; GitHub adapter emits no emails/phones. |
| Validation failures are tested | `tests/test_validation.py`. |
